Ask me for the feature name first and use it as a hint.

Then create a concise checklist of what we did for that feature.

Checklist requirements:
- Group items by area when possible (backend, frontend, UX if needed).
- Use clear, action-focused checkboxes(high level as a release log).
- Use only bullet points for lists and also for sub-items when needed, but keep the main checklist items concise.
- Give just the checklist without any additional commentary or explanations.
- Use formatting compatible to use in discord or markdown files.
- Keep each item specific and verifiable.

Output format:
## Feature
[feature name]
